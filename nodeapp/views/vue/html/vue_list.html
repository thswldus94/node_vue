1. v-for 엘리먼트 배열 매핑하기

v-for로 배열 리스트 렌더링을 할 수 있습니다~

item in items 같은 형식으로 php foreach 형식과 비슷하네용 (item of items 도 괜찮아요 of도 쓸수있다는점 ㅎ.ㅎ)


  • {{ item.message }}


v-for는 부모 범위 속성에 대한 모~든 권한이 있어유. 또한 인덱스 요소(두번째)까지 사용이 가능해요. 인덱스는 늘 그렇듯.. 0번부터 ^.^

  • 부모 속성: {{parentMessage}} / 인덱스: {{index}} / 아이템메시지: {{item.message}}

2. v-for와 객체

배열뿐만 아니라 객체의 속성도 반복할 수 있수요!

Key값과 index까지 전달 하려면 v-for="(value, key, index) in object" 와 같이 해주면 돼요. 두번째 인자가 key, 세번재 인자가 index값이예요

  • index: {{ idx }} / key: {{ key }} / value: {{ val }}

3. v-for의 Key

만약 리스트 데이터에서 순서가 변경된 경우? DOM요소를 직접 이동하여 수정하지 않아요

대신 Vue는 각 요소를 적절한 위치에 패치하고 렌더링 할 내용을 반영할지 확인해요

각 노들을 추적하고 기존 엘리먼트의 재사용,정렬 하기 위해 각 항목들의 key를 확인할 수 있게 key속성을 사용하는 것이 좋아요.

4. 배열변경감지

감시중인 배열의 변이 메소드를 래핑하여 뷰 갱신을 트리거 합니당! 다음은 래핑된 메소드예요

  • {{ method.name }}

app55.methods.pop({name:'splice()'}); 뭐 이런식으로 뽑아내거나 추가하거나 할 수 있어요.

5. 배열 대체

변이 메소드는 원본 배열을 변형하는데, 원본 변형을 원치 않는 경우 아래의 세 메소드를 사용하면 항상 새 배열을 반환해요

  • {{ method.name }}

근데 이러면 Vue가 기존 배열을 버리고 다시 렌더링 할줄 아는데?!! 그건 아님 ㅋ 뭐.. 대체해서 사용한다구 함

6. 주의사항!

Javascript의 제한으로 인해 Vue는 아래와 같은 상황을 감지할 수 없어요

1. 인덱스로 배열 항목을 직접 수정 vm.items[2] = { 'name' : 'new name!!!' }

2. 배열 길이를 수정하는 경우 vm.items.length = 1


1번의 경우 Vue.set(vm.items, 인덱스번호, 새로운 값); 순으로 실행해야 합니다.

2번의 경우 vm.items.splice(새로운 값)

7. 필터링/정렬 된 결과 표시

원본 데이터를 변경하지 않고 배열의 필터링 된 버전이나, 정렬된 버전을 표시할 때 사용합니다.

v-for="n in evenNumbers"로 짝수 넘버만 보여주도록 해볼게요
