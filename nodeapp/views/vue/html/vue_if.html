1. v-if

말 그대로 if문을 사용할 수 있어요

ok인 경우 Yes가 나타납니다!

else 조건으로 맞을 시 No가 나타납니다!


쉽게 말해 가상의 엘리먼트를 만들 수도 있습니다

하나의 엘리먼트에만 추가가 가능하지만 여러개를 추가하고 싶을 수도 있잖아요??

그럴때 template 엘리먼트를 사용합니다. 물론 이건 나중에 렌더링된 결과에는 나오지 않아요~


2. v-else

v-else 디렉티브는 반드시 v-if 뒤에 위치해야 합니다! v-else-if도 있어요~

랜덤숫자가 100보다 큰 경우 내가 보일거예요! 랜덤숫자가 50보다 큰 경우 내가 보임!! 이제는 안보입니다!

3. Key를 이용하여 재사용 가능한 엘리먼트 제어

재사용 가능한 엘리먼트 제어가 쉬워요. 예를들어서 로그인 할 때
아이디를 넣는지, 이메일을 넣는지 구분할 수 있는 코드를 짜볼게요.
단순히 key를 사용하지 않는 다면 두 개는 동일한 엘리먼트로 사용하기 때문에
인풋 안의 내용이 지워지지 않아요. 그럴때 이건 유일하다!! 라는걸 알려주기 위해 key를 씁니다
유일한 id같은 개념이죠. 재사용 하지 않고 그린답니다.


4. v-show

v-show는 단순히 toggle느낌입니다. display 속성 토글 수준?
처음부터 계속 렌더링 되고 DOM에 남아있기 때문에 template 구문과 else, if 구문도 동작하지 않습니다.

v-if는 레알 조건부 렌더링 입니다. 대신 렌더링 초기에 조건이 false인 경우 아무것도 안합니다
망할놈이 일도 안하는거죠;; true가 나올 때 까지 일을 안합니다
대신 v-show는 처음부터 무조건 렌더링됩니다
바뀌는 빈도가 많으면 v-show, 적으면 v-if 가 쓰기 좋습니다.
